## Detailed Competitor Profiles

### 1. Bloom (bloomapp.com)

**Positioning**: "Learn about money by making decisions" — interactive branching scenario financial education
**Target Users**: Gen Z and young millennials (18-30) who learn through interactive stories, not lectures

**Key Features**:
- **"Bloom Stories"**: Interactive branching narratives where users make financial decisions and see consequences — "You just got a NT$10,000 bonus. Do you: A) Invest in TSMC, B) Put it in savings, C) Spend it?" Each choice leads to a different story outcome with plain-language explanations of what happened historically
- **"Money Personality" Quiz**: AI-generated financial personality assessment — "You're a Cautious Explorer: you like to understand before you invest" — with personalized learning path
- **"Daily Bite"**: One interactive financial concept per day, delivered as a 2-minute story with a decision point
- **"Scenario Library"**: 50+ real-world financial scenarios based on historical market events — "What would you do if the market dropped 20% tomorrow?" with historical context
- **"Progress Map"**: Visual map of financial concepts mastered, with branching paths showing what to learn next
- **"Bloom AI Coach"**: Conversational AI that adapts to user's knowledge level — asks questions, provides hints, explains concepts in context

**UX/Design Approach**:
- **Interactive fiction format**: Financial education delivered as choose-your-own-adventure stories
- **Mobile-first**: Native app with swipe-based interactions
- **Gamified**: Points, levels, and story completion badges
- **Personalized**: AI adapts story difficulty and topics to user's interests and knowledge gaps
- **Visual**: Illustrated characters and scenarios, not charts and tables

**Unique Capabilities**:
- **Branching scenario learning**: No other competitor uses interactive fiction for financial education — this is a unique pedagogical approach
- **Decision-consequence modeling**: Users see the historical consequences of different financial decisions — "If you invested in TSMC 5 years ago, you'd have X"
- **Personalized learning path**: AI maps user's knowledge gaps and creates a custom learning journey

**Comparison with Stock Explorer**:

| Feature | Bloom | Stock Explorer |
|---|---|---|
| Interactive Scenarios | ✅ Branching stories | ❌ Not built |
| AI Personalization | ✅ Adaptive coach | ❌ Not built |
| Decision-Consequence | ✅ Historical modeling | ❌ Not built |
| Plain-language | ✅ Core feature | ✅ Core feature |
| Company Analysis | ❌ Concept-focused | ✅ Deep coverage |
| TW Market | ❌ US/Global | ✅ Deep |
| Story First | ✅ Interactive stories | ✅ Linear stories |
| Gamification | ✅ Points + levels | ❌ Not built |
| Mobile App | ✅ Native | ❌ Streamlit only |

**Key Insight for Stock Explorer**: Bloom's "branching scenario" approach is a fundamentally different pedagogical model than Stock Explorer's linear narrative. Instead of "here's what happened to TSMC," Bloom asks "what would YOU do in this situation?" This active learning approach is more engaging and memorable. Stock Explorer's "historian" positioning could adopt a lighter version: "What would have happened if..." scenarios that let users explore historical decision points.

## New Feature Gaps Identified

### [ISSUE-C81] "What Would Have Happened" — Historical Decision Scenario Explorer

**Source Competitor**: Bloom (branching scenario stories), 長投學堂 (historical case studies), Visual Capitalist (animated data stories)
**Priority**: P2
**Effort**: 14-20h
**Alignment**: Core value #1 "Story first, data second" + Core value #4 "Point-to-point knowledge construction" + "Historian" positioning

**Description**: Bloom's branching scenarios let users make financial decisions and see historical consequences. 長投學堂's historical case studies show what happened during specific market periods. Stock Explorer has an event dashboard (M5) that lists events but doesn't explore "what would have happened if..." scenarios. A "What Would Have Happened" feature would: (1) present 5-10 historical decision points for each company — "TSMC's stock dropped 30% in 2008. What happened to investors who held vs. sold?" (2) show the historical outcome of each decision with plain-language explanation, (3) frame as educational exploration — "Here's what happened historically. Your decision would have led to X." (4) connect to the "historian" positioning — this is literally history, not prediction.

**User Value**: Beginners learn best through "what if" scenarios. Instead of reading about TSMC's history, they explore "What if I invested in TSMC before the 2008 crisis?" — and see the actual historical outcome. This transforms passive reading into active exploration.

**Implementation Notes**:
- Create `src/data/historical_scenarios.yaml` with 5-10 decision points per top stock
- Each scenario: decision point → historical outcome → plain-language lesson
- Display as interactive cards on the company page: "🤔 歷史情境：如果..."
- Use existing FinMind data to calculate historical outcomes
- Frame as "This is what happened" not "This is what to do"
- Start with top 10 TW stocks, expand based on engagement

**Competitive Gap**: 🔴 No TW competitor has historical decision scenarios. Bloom proves demand for interactive scenarios internationally. This is the ultimate "historian" feature — users literally explore history.

### [ISSUE-C82] "Animated Data Story" — Scrollable Visual Company History

**Source Competitor**: Visual Capitalist (animated data stories, infographic-first), 長投學堂 (historical case studies)
**Priority**: P2
**Effort**: 12-16h
**Alignment**: Core value #1 "Story first, data second" + Core value #2 "PPT-style presentation" + "Ten-second test"

**Description**: Visual Capitalist's animated data stories show how metrics changed over time through interactive scrolling. Stock Explorer's charts are static Plotly visualizations — they show data but don't tell a story. An "Animated Data Story" would: (1) add a scrollable visual timeline to each company page — as the user scrolls, charts animate to show how key metrics changed over time, (2) each scroll step reveals one data point with plain-language context — "2018: Revenue was NT$1.0T. That's like the entire GDP of Taiwan's retail sector." (3) the animation is driven by scroll position — creating a "story unfolds as you scroll" experience, (4) this is a more visual version of C34 (Company Story Timeline) that focuses on data visualization rather than text narrative.

**User Value**: Static charts require interpretation. Animated data stories guide the user through the data, revealing one insight at a time with plain-language context. This makes complex financial data accessible to beginners who don't know how to read charts.

**Implementation Notes**:
- Use Plotly's animation framework or a JavaScript scroll-driven animation library
- Create a "📊 數據故事" section on the company page
- Each scroll step: one chart + one plain-language insight
- Metrics to animate: revenue, profit, P/E, dividend, institutional ownership
- Start with a simple implementation: 5 scroll steps per company, static images that fade in (MVP)
- Enhance to full animation in a later iteration

**Competitive Gap**: 🟡 Visual Capitalist has animated data stories but not for individual TW companies. No TW competitor has scrollable animated company histories. This would be a unique visual differentiator.

### [ISSUE-C83] "Investment Memo Template" — Structured Reflection Tool

**Source Competitor**: 長投學堂 (投資備忘錄 template), Tykr (stock stories), Tastytrade (trade journal)
**Priority**: P2
**Effort**: 6-10h
**Alignment**: Core value #1 "Story first, data second" + Core value #4 "Point-to-point knowledge construction" + "Historian" positioning

**Description**: 長投學堂's "Investment Memo" is a structured template for recording investment analysis — "Why I'm interested in this company" with specific prompts. ## Key Insights from Round 16

### 1. **Interactive Scenarios Are the Next Frontier in Financial Education**
Bloom's branching scenario approach is fundamentally different from all previous competitors — instead of telling users what happened, it asks "what would you do?" This active learning model is proven in other education domains (medical training, military strategy) but barely explored in financial education. Stock Explorer's "historian" positioning is perfect for this: "Here's a historical decision point. What would you have done? Here's what actually happened."

### 2. **Personality and Emotional Engagement Drive Retention**
Cleo (sassy AI personality), Bloom (gamified stories), and Plum (behavioral nudges) all use emotional engagement as a core retention mechanism. Stock Explorer's warm but serious tone is appropriate for the "historian" positioning, but there's room for more personality — especially in micro-copy, quiz feedback, and daily engagement content. The C66 (Conversational Tone) and C60 (Concept Mastery Badges) features partially address this.

### 3. **TW Pure Education Platforms Are Underserved**
長投學堂 is the only major TW pure education platform (no broker affiliation) focused on long-term investing. Stock Explorer is another — but there's room for more. The TW market has many broker-integrated education features but few independent, pure-education platforms. Stock Explorer's "historian" positioning fills this gap.

### 4. **Visual Storytelling Is Becoming Table Stakes**
Visual Capitalist (infographic-first), 長投學堂 (historical case studies), and Bloom (illustrated stories) all use visual storytelling as their primary education method. Stock Explorer's PPT-style is aligned with this trend but could be more visual — fewer words, more diagrams, more animation. C82 (Animated Data Story) would close this gap.

### 5. **Self-Assessment Drives Personalized Learning**
Bloom (Money Personality), Cleo (Money Moods), and Plum (behavioral insights) all start with self-assessment — understanding the user before teaching them. Stock Explorer has no self-assessment — it assumes all users are the same. C85 (Financial Wellness Check) would add a personalization layer that makes all other content more relevant.

### 6. **Historical Case Studies Are a Unique "Historian" Feature**
長投學堂's historical case studies and Bloom's branching scenarios both use historical events as educational content. This is the purest form of "historian" education — teaching through history, not through prediction. Stock Explorer's event dashboard lists events but doesn't create immersive case studies. C84 (Market Event Case Study) would be the ultimate "historian" feature.
