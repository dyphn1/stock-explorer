# User Feedback — Stock Explorer

> **Priority**: URGENT — Always processed before roadmap items.

## How it works
1. Daniel (or any user) drops feedback files here
2. PM reads ALL feedback on every cron run
3. PM analyzes and adds to roadmap as highest priority
4. PM assigns to appropriate agents

## File naming
Format: `YYYY-MM-DD_<type>_<brief-description>.md`

Types:
- `bug` — Something is broken
- `feature` — New feature request
- `ux` — UX/UI issue or suggestion
- `perf` — Performance issue
- `other` — Anything else

Example: `2026-06-20_ux_sidebar-navigation-confusing.md`

## Template
```markdown
# [Type]: [Brief Description]

**Date**: YYYY-MM-DD
**Reporter**: Daniel

## What's the problem?
[Describe the issue or request]

## Expected behavior
[What should happen instead]

## Screenshots / Context
[If applicable]
```
