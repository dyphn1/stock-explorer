# ADR-010: PPT 風格設計原則

## 狀態
已接受

## 日期
2026-06-08

## 背景

Stock Explorer 的目標用戶是新手投資人。傳統股票工具的數據密集表格和大量文字對新手不友善。

## 決策

採用 **PPT 風格**作為核心設計原則：

1. **一頁一重點**：每個頁面只傳達一個核心訊息
2. **圖片主導，文字輔助**：圖表佔 > 60% 頁面面積
3. **十秒測試**：新手能在 10 秒內複述核心概念
4. **漸進式揭露**：預設只顯示關鍵數據，細節按需展開

## 具體規範

### 佈局
- 每個頁面最多 1 個主要圖表
- 圖表上方：一句話標題
- 圖表下方：白話說明 + 生活化比喻
- 禁止在一頁中堆疊 3+ 個圖表

### 數據卡片
- 每個卡片：標題 + 大字數據 + 白話說明
- 最多 4 個卡片一行（桌面版）
- 使用 `_白话_card()` 共享元件

### 文字限制
- 卡片說明文字 ≤ 200 字
- 分析文字 ≤ 400 字
- 超出時使用 `st.expander` 折疊

## 理由

1. **新手友善**：減少認知負擔
2. **記憶點**：一頁一個重點更容易記住
3. **差異化**：與財報狗等數據密集工具形成差異

## 後果

- ✅ 新手更容易理解
- ✅ 頁面更清爽
- ⚠️ 需要嚴格執行，否則容易退化回數據密集風格
- ⚠️ 部分頁面需要重寫以符合規範
